控制器 Controller・開卡

Controller Architecture & Bring-up
0/7

控制器(controller)是 SSD / 記憶卡的大腦。FAE 要先有一張「資料怎麼從 Host 流到 NAND」的方塊圖,才看得懂後面所有 debug。

FAE 要掌握的重點

  • 資料路徑 vs 控制路徑:讀寫資料走 ECC/buffer,命令/設定走 CPU/暫存器。出問題先判斷是哪條路。
  • FTL:邏輯→實體位址映射、garbage collection、wear-leveling —— 效能與壽命都在這。
  • ECC/LDPC:NAND 一定有 bit error,靠 LDPC 糾正;RBER 太高會掉到 read-retry / 解碼失敗。
  • 多通道 / 多 CE:並行存取多顆 die 才有高頻寬(見 interleave)。
  • Regulator、OSC、VDT 屬基礎設施,下一項「Power Domain」會細談。

控制器內不同區塊用不同電壓、可獨立開關的「電源域(power domain)」。外部電源經內部 Regulator 降出各域電壓;省電時可關掉用不到的域。FAE debug 掉電、開不了機、耗電異常,都要先看這張圖。

FAE 常見應用

  • 量產卡開不了機:先量各域電壓有沒有正常建立、順序對不對。
  • 掉電保護(power loss):VDT 觸發後韌體要在電容撐住的時間內把關鍵 metadata 寫回。
  • 省電:idle/sleep 時關閉不用的域,與下面「Power Consumption」直接相關。

量測與工具 Instruments

Measurement & Analysis Tools
0/7

主機介面靠時脈(CLK)對齊命令/資料。資料必須在時脈取樣邊緣前穩定一段(建立 tSU)、後維持一段(保持 tH),取樣才正確。FAE 用 LA / Protocol Analyzer + 示波器量這段 margin。

FAE 實作

  • 用 Protocol Analyzer 解命令層(CMD/RESP/UPIU),用示波器 / LA 看實體層 timing 與眼圖。
  • 量 tSU/tH margin、看是否需要調 host 端 driving strength / delay line(timing training)。
  • 對照各介面 SPEC(SD、eMMC、UFS)的 timing 表逐項驗證。

NAND 介面用一組控制訊號把匯流排上的 byte 分成命令 / 位址 / 資料三種相位。LA(邏輯分析儀)抓下這些訊號後,FAE 要會「parsing」—— 把波形翻譯回 命令序列,才能定位讀寫錯誤。

FAE 實作

  • 用 LA 設好各訊號,觸發在特定命令(如 erase 60h-D0h),抓出問題前後的完整序列。
  • 對照 NAND datasheet 的 timing(tWP、tWC、tRC、tDS/tDH…)確認是否合規。
  • 高速模式量 DQ/DQS 眼圖,連到示波器頁的眼圖 / 時序裕度觀念。

持續高速讀寫會發熱。當溫度逼近上限,韌體會主動降速(降頻、限 IOPS)把溫度壓在安全門檻下 —— 這就是 thermal throttling。代價是持續吞吐量掉下來,表現為「先快後慢」。

FAE 重點

  • 短時間 benchmark 看不到 —— 一定要長時間 / 持續負載才會觸發。
  • 受散熱條件影響大:有無散熱片、機殼、環境溫度(配合 Chamber 測)。
  • 與 Power Consumption 互為因果:功耗→發熱→throttle。

各狀態的耗電都要量:idle、read、program / erase 的電流尖峰、以及 sleep 低功耗態。Host SPEC(eMMC/UFS/SD)對平均電流與峰值都有上限。FAE 用 Data Logger / 電源分析儀量 current vs time。

FAE 量測要點

  • 取樣率要夠高才抓得到 program/erase 的瞬間尖峰(否則被平均掉)。
  • 分狀態量:active(R/W)、idle、sleep,各算平均 + 峰值。
  • 對照 Host SPEC 的 ICC 限制;超標會被客戶退。配合 Thermal 一起看。
