01

NAND vs NOR：兩種 array 的根本取捨

Flash 的所有 cell 都是「可儲存電荷的 MOSFET」，差別在 array 怎麼接線。這個接線方式決定了存取速度、cell 面積、成本，進而決定它適合做 code storage 還是 mass data storage。

表 1.1 · NAND vs NOR 特性對照
面向NOR FlashNAND Flash
Array 結構並聯，cell 直接接 BL串聯成 string，共用接觸
Cell size大 (~10 F²)小 (~4 F²)
Random read快 (~60–120 ns)慢 (~10–50 µs / page)
Sequential write慢 (byte/word)快 (page，數 KB 同時寫)
定址單位byte / word，可隨機讀page 讀寫、block 抹除
XIP (code 就地執行)可，適合 code storage不適合，需先載入 RAM
成本 / bit高最低（半導體記憶體中）
典型用途BIOS / firmware / MCU codeUSB、SD、SSD、eMMC/UFS mass storage
◆ Senior 觀點

NAND「不能 XIP」不是缺點而是設計取捨的結果：串聯使單一 cell 無法被獨立量測電流，必須整條 string 導通才讀得到目標 cell。這也是為什麼 NAND read 一定要對 unselected WL 加 Vpass 把它們「無條件導通」——這個 Vpass 正是後面 read disturb 的源頭。架構上的取捨，會一路傳遞到 reliability。

02

Flash 應用介面：從 USB 到 PCIe SSD

同樣的 NAND die，套上不同的 controller 與 host 介面，就變成不同產品。介面演進的主軸是頻寬與協定複雜度：從 USB / SD 的可移除消費級，到 eMMC / UFS 的嵌入式，再到 SATA / PCIe 的儲存級。

表 2.1 · 六大 Flash 應用介面比較
介面定位 / 形態頻寬等級優點缺點
USB Flash可移除隨身碟USB 2.0 ~480 Mb/s；3.x 可達數 Gb/s 即插即用、通用、便宜效能與壽命管理弱、controller 簡單、隨機效能差
SD / microSD可移除卡（相機、手機）UHS-I ~104 MB/s；UHS-II / Express 更高 標準化、體積小、生態成熟耐久度 / 隨機 IOPS 受限、卡務等級差異大
eMMC嵌入式封裝（BGA 焊在板上）HS400 ~400 MB/s 整合 controller、低成本嵌入式、省空間並列匯流排、頻寬天花板低、半雙工
UFS嵌入式（取代 eMMC）UFS 3.1 ~2.1 GB/s（雙 lane） 序列差動、全雙工、支援 command queue、低延遲成本較 eMMC 高、設計複雜
SATA SSD2.5"/M.2 用 AHCISATA III ~600 MB/s（飽和） 相容 HDD 介面、好換裝、成熟穩定頻寬已被介面卡死、AHCI 單佇列、延遲高
PCIe / NVMe SSDM.2 / U.2 / add-in cardGen4 x4 ~7 GB/s；Gen5 更高 直連 PCIe、NVMe 多佇列 (64K queues)、極低延遲、高 IOPS功耗 / 發熱高、成本高、需散熱設計
▣ 機制：為什麼介面一直往「序列差動 + 多佇列」走

並列匯流排（eMMC、傳統 NAND async）在高頻下受 skew 與訊號完整性限制，頻寬難拉高；UFS / PCIe 改用序列差動點對點，並用 command queue 讓多個 outstanding command 並行，才能餵飽底層多 plane / 多 die 的平行度。換句話說：介面的演進，是為了把 NAND 內部本來就存在的平行度釋放出來。

◆ Senior 觀點

同一顆 die 在不同產品的「實際壽命」差很多，因為 controller 的 FTL（wear leveling、GC、over-provisioning、ECC/LDPC 強度）不同。做 FA 時若只看 die 而忽略 controller 策略，會把「系統層 write amplification 造成的早夭」誤判成「cell 本質 endurance 不足」。

03

三大操作：Erase / Program / Read

NAND 的三個動作都靠調整 cell 內儲存電荷量來改變 threshold voltage（Vth）。記住一條總綱：抹除把電荷拉走（Vth 降）、寫入把電子打進去（Vth 升）、讀取則是在 Vth 軸上量它落在哪一邊。

3.1 Erase — block 單位、FN tunneling

抹除以 block 為單位。2D / 3D body erase 直接把 channel/well 拉到 ~20V、WL 接 0V，靠強電場把儲存層的電子 FN tunneling 拉回 channel，Vth 降到最低的 erase state。3D 因 string 與基板解耦，常用 GIDL erase：在 source/drain junction 用 GIDL 產生 electron–hole pair，把 holes 灌進 channel 抬高 body 電位。

3.2 Program — ISPP 與 self-boosting inhibit

寫入以 page 為單位，採 ISPP（Incremental Step Pulse Program）：一連串逐步升高的 Vpgm 脈衝 + program-verify，把 Vth 精準推到目標 state。要寫的 BL 接 0V 讓 channel 低電位形成強場注入；不寫（inhibit）的 BL 拉高，靠 self-boosting 把 channel 電位浮高、壓低跨 tunnel oxide 的電場避免誤寫。

⚠ 易錯陷阱：program 順序

NAND 必須從 source 端往 drain 端依序寫（WL by WL），因為一個 cell 的 Vth 量測會受其與 source 之間 cell 狀態影響（background pattern dependency）。順序錯了，verify 階段與真正 read 階段看到的 Vth 會不一致，直接造成 read fail。

3.3 Read — Vr 掃描與 sensing

讀取對選中 WL 給參考電壓 Vr，其餘 WL 給 VpassR(~7V) 無條件導通。若 cell Vth < Vr → string 導通（讀為 1）；Vth > Vr → 不導通（讀為 0）。多階 cell（MLC/TLC/QLC）就用多個 Vr 把 Vth 軸切成多段。

3.4 SGD Vth 如何影響操作窗

SGD（drain select gate）本身就是一顆 transistor，它的 Vth 必須落在一個剛剛好的 operating window，否則 program/inhibit 都會壞。這是新人最容易忽略、senior 最先檢查的地方。

◆ Senior 觀點：SGD Vth 的兩條 failure tail

低 Vth 尾（leakage 端）：SGD Vth 太低，在 inhibit 時關不乾淨，subthreshold leakage 會把已 boost 起來的 channel 電位洩掉，導致 inhibit 失效而誤寫。關鍵直覺：channel capacitance 很小，所以這裡是靠「漏電」就足以崩掉 boosting，不需要 SGD 完全導通。
高 Vth 尾（floating 端）：SGD Vth 太高，selected string 在 program 時 SGD 開不夠、channel 與 BL 脫離（floating），0V 傳不進去 → 跨 tunnel oxide 的 FN 場不足 → 寫不進、program 變慢或失敗。

04

Reliability Issues

Reliability 的共同語言是 Vth distribution 的位移與展寬：任何機制只要讓 distribution 漂移或變寬到吃掉 read margin，就會產生 bit error。下面四個是 FA 最常碰到的。

4.1 Retention（資料保存）

寫好後電荷隨時間漏失，programmed state 的 Vth 往下漂、distribution 展寬。在 charge-trap（CTF）cell，short-term retention 由幾個機制主導：lateral migration（LM，橫向遷移）、vertical relaxation（VR，縱向鬆弛）、trap-assisted tunneling（TAT，去陷阱），以及 BE-tunnel oxide 內近 channel trap 的快速 detrapping。

4.2 Cross-temperature read & Arrhenius 外推陷阱

「寫入溫度」與「讀取溫度」不同時，Vth 會整體偏移（cross-temperature read），因此 read level 需要溫度補償。更深的坑在 retention 壽命的溫度外推。

✖ 高風險陷阱：別用單段 Arrhenius

3D charge-trap cell 的 Vth shift 與烘烤溫度呈非線性（兩段 Ea），這與 2D FG 的線性行為不同。直接套用幾十年來的單一溫度加速模型，會把低溫實際壽命嚴重高估。低於 ~90 °C 的 retention 必須用較低溫、長時間（至少數週）資料外推驗證，不能只靠高溫 bake。這是 reliability methodology 必須明確標注的一條。

4.3 Read disturb

讀取時，被加 VpassR 的 unselected WL 上的 cell 受到弱電場，少量電子被注入儲存層 → Vth 緩慢往上漂。特性：erased（負 Vth）cell 最敏感、隨 read 次數與 Vpass 升高而惡化、distribution 往高 Vth 展寬；屬可恢復（erase 後重寫即還原，非永久 oxide 損傷）。緩解靠 ECC/LDPC、Vth window margin、read refresh。

4.4 Program disturb（含 3D 新模式）

寫某 page 時，未被選中的鄰近 cell 受到 Vpgm / self-boosting 電場干擾。2D 主要是同 WL 上 inhibit cell 的誤寫；3D 因一條 BL 接多個 SGD（DSL_1..N），多出 Y / XY 兩種新模式——其中 BL 接地、DSL off 的 Y mode 最棘手，因為 3D 的 DSL leakage 較大，boosting 電位可能透過 DSL 漏掉。

▣ 反直覺：CTF 的 program disturb 可能是 Vth「下移」

新人直覺會以為「disturb 一律讓 Vth 升」。但在 3D charge-trap cell，強 channel boosting 下的 inhibit cell 可能發生soft erase——boost 起來的高 channel 電位反而把電子拉出儲存層，造成 Vth downshift。這與 read disturb 的「Vth 上移」方向相反，FA 時若看到 inhibit 區 Vth 下移，要往這個方向想，而不是排除 program disturb。

4.5 Endurance 與 cell-to-cell interference

P/E cycling 會在 tunnel oxide 累積 trap（trapping/detrapping），使 Vth window 漂移、SILC 增加、retention 變差，最終達到 endurance 上限。另外 CTCI（cell-to-cell interference）即使在 3D 改用 SiN 儲存層後仍存在：鄰近 WL 儲存的電子仍會影響本 cell 的 channel 電位，使 distribution 展寬——這是 distribution 工程要持續對抗的雜訊源。

表 4.1 · 四大 disturb / 漂移的方向與對策
機制受影響 cellVth 方向是否永久對策
Read disturb同 block 未選 WL（erased 最敏感）↑ 上移否（可恢復）ECC、read refresh、Vpass 優化
Program disturb鄰近未選 cell（3D：Y/XY）↑ 為主；CTF 可 ↓否self-boosting、program 順序、verify
Retention已寫入 cell↓ 下移 + 展寬否（電荷漏）read level 補償、refresh、cell 結構
Endurance / cycling全 cell（tunnel oxide trap）window 漂移是（趨永久）wear leveling、強 ECC、限 P/E
05

2D vs 3D NAND

2D NAND 靠微縮平面 cell 來增密度，逼近 ~1x nm 後 floating gate 之間的耦合、電子數統計漲落（RTN）讓 distribution 失控，微縮走到盡頭。3D 改變思路：不再縮平面，而是往上堆疊 WL，用垂直 channel pillar 串起整條 string。

表 5.1 · 2D vs 3D 對照
面向2D NAND3D NAND
密度增長縮平面線寬（已撞牆）堆疊層數（128 / 176 / 200+ 層）
主要儲存方式多為 floating gate多為 charge trap（亦有 FG 版）
Gate 結構平面 gateGate-all-around（環繞）
Cell 間耦合嚴重（鄰近 FG 耦合）改善但 CTCI 仍在
RTN / 電子統計小 cell 漲落大cell 較大、較緩
製程挑戰微影解析度high-aspect-ratio channel hole 蝕刻
◆ Senior 觀點：GAA 的雙重好處

Gate-all-around 環繞 channel 帶來兩個物理好處：(1) 更佳的 subthreshold slope 與更低 leakage；(2) 因曲率半徑小，tunnel oxide 內電場更高，program/erase 效率提升。但代價是 channel hole 的 aspect ratio 隨層數爆增——這正是 3D 良率與製程的主戰場。

06

Charge Trap vs Floating Gate

兩者差別在電荷儲存在哪：FG 把電荷存在導電的 poly-Si 浮閘（電荷可在 FG 內自由移動、cell 間需物理隔離）；CT（SiN/SONOS）把電荷陷在絕緣的氮化矽 trap 裡（電荷不流動，儲存層可連續共用）。

表 6.1 · Floating Gate vs Charge Trap
面向Floating GateCharge Trap (SiN)
儲存層導電 poly-Si 浮閘絕緣 SiN，電荷陷於 trap
電荷行為可在 FG 內自由移動不流動、局部釘住
cell 隔離須物理切開（製程複雜）可連續共用、memory hole 直徑大、利於堆高
缺陷容忍單一漏電路徑→整個 FG 電荷流失單缺陷只損局部 trap 的電荷
耦合 / 干擾FG 間耦合強較佳，但仍有 CTCI
Short-term retention相對穩定較弱（LM/VR/TAT 快速 detrap）
3D 採用度較少（製程難）主流（RG NAND / SONOS）
▣ 機制：CT 怎麼補救 retention 弱點

CT 的快速 detrapping 用 band-engineered (BE) tunnel dielectric 改善：erase 時增強 hole injection，同時用 high-k / metal gate 抑制電子從 gate 端 back-tunneling。理解這點，才能在 FA 看到「短期 retention 異常」時，把矛頭指向 BE-tunnel / blocking 層的製程，而不是泛泛歸因「電荷漏掉」。

07

3D Flash 製程簡介

3D NAND 的精神是「一次蝕刻打穿所有層」：先堆好多層薄膜，再用單次（極困難的）high-aspect-ratio 蝕刻一次定義所有垂直 cell，把每層的關鍵 mask 從「每層一次」降為「全部一次」，這正是成本優勢來源。以 charge-trap RG NAND 的 gate-replacement 流程為主軸：

  • Channel hole 蝕刻是良率瓶頸：層數越高、AR 越大，孔形（彎曲、上下直徑差）直接影響各層 cell 一致性。
  • Staircase：每層 WL 都要拉出接點，做成階梯狀，是面積與微影的額外成本。
  • CMOS 配置：CoA（旁邊）→ CuA（CMOS under Array，把週邊電路塞到 array 下方）→ WoW（wafer-on-wafer，CMOS 與 array 分開做再鍵合），都是為了塞進更多 plane、拉高平行度與面積效率。
◆ Senior 觀點

很多「沿 WL 方向（pillar 上下）的 Vth 系統性差異」其實是 channel hole 直徑隨深度漸縮造成的——上下 cell 的 gate-channel 幾何不同。看到 layer-dependent 的 Vth 偏移，先想製程幾何（hole profile），再想 array operation。把 device 行為對應回製程，是 FA 的核心能力。
