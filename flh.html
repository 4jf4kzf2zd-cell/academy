Mental model 00

兩個頻寬世界:這是整本手冊的地基

新人最常見的混淆,是把「NAND 跑 2400 MT/s」這種數字,跟「NAND program 一個 page 要多久」混為一談。它們屬於兩個彼此獨立的頻寬世界。把這條界線刻進腦袋,後面所有題目都會自動歸位。

FLH (array) bandwidth — 真實媒體速度

指 cell array 真正能 program / read 的吞吐量,由 cell 物理決定:

# 媒體吞吐量
program BW = page size / tPROG
read    BW = page size / tR

tPROG 受 ISPP step 數、FN tunneling 效率、Vth distribution 收斂速度限制(TLC/QLC 動輒數百 µs ~ ms);tR 受 sensing time、WL 的 RC delay、多 pass read 影響。單顆 die 單 plane 的 FLH BW 很低,通常只有數~數十 MB/s 等級。

IO (interface) bandwidth — 介面搬運速度

指 controller ↔ NAND 之間 DQ bus 的傳輸速率,純粹由介面決定:

# 介面頻寬
IO BW = DQ width × DDR data rate(MT/s)
# 例:×8 @ 2400 MT/s
     = 1 byte × 2400 MT/s = 2400 MB/s ≈ 2.4 GB/s

資料只在 cache register ↔ controller 之間搬移,完全不碰 cell。提升靠介面工程(DDR rate↑、ODT、降 Vccq、減 channel loading),這是「資料搬多快」。

機制重點

FLH BW 由 cell 物理定,IO BW 由 periphery 電路定。單顆 die 的 FLH BW ≪ IO BW。所以系統不是靠單顆衝快,而是靠 interleave + cache 把慢的 array 藏在快的 IO 後面 —— 這就是後面第 07、08 章的全部主題。

Senior 思維

看到任何「頻寬不如預期」的問題,第一個動作不是查 spec,是先問:這是 array-bound 還是 IO-bound? 兩者的 root cause、量測手段、對策完全不同(第 11 章會給 decision tree)。把問題歸錯邊,後面全錯。

Architecture 01

架構速覽,與 cache register 的解耦魔法

NAND 的 datapath 是一條清楚的流水線。理解「資料在哪裡被誰搬」,比背 die 結構更重要。

圖 1. IO 對著 cache register toggle,從不直接碰 cell。所以 array 速度與 IO 最高速度是兩個獨立電路 domain —— 一顆慢 program 的 QLC,仍可掛高速 ONFI 介面。

由下而上的單元:cell → string(BL 方向串聯)→ page(同一條 WL)→ block(共用 string、最小 erase 單位)→ plane → die。一顆 die 內常有 2~4 個 plane,可同時動作(multi-plane,第 07 章)。

機制重點

Page buffer / cache register 是整個架構的關鍵解耦點。program 時資料先灌進 register、再由 array 慢慢寫;read 時 array 先 sense 到 register、再由 IO 快快搬走。沒有這個 register,慢 array 會直接拖死快 IO,所有 interleave / cache 技巧都不可能存在。

Interface 02

從 Legacy async 到 Toggle / ONFI DDR

介面演進的唯一動機:page 越做越大,array 內部吞吐量追不上,必須把資料搬運加速以重建平衡。40 MB/s 的 async 介面在 1KB page 時代剛好,到 4KB/8KB page 就嚴重失衡,於是 DDR 介面登場。

三種介面的本質差異

Legacy ASYNCWE# / RE# 脈衝SDR · ~40 MB/s—
ONFI NV-DDR (SSI)WE#→CLK, RE#→W/R#DDR · source-syncDQS
Toggle-Modecmd/addr 用 WE#;data 用 DQSDDRDQS
機制重點 · Toggle 的核心

command / address 仍由 WE# 以 SDR 方式 latch(legacy 風格),但 data 改用 DQS strobe、雙邊緣(DDR)傳輸。Toggle 只多了一支 DQS、沒有 free-running CLK;ONFI 則把 WE# 變成 CLK、RE# 變成 W/R#,是完整的 source-synchronous。看波型時,先確認你面對的是哪一版,latch 機制完全不同。

圖 2. 左:命令位址走單邊緣 SDR(WE# latch)。右:資料走雙邊緣 DDR(DQS strobe),這是 Toggle/ONFI 倍速的來源。
Bandwidth 03

「1600 / 2400 MT/s」到底是哪一塊?

這是 IO bandwidth,不是 array。指的就是上一章 DDR 那段 —— DQ bus 被 DQS strobe 的傳輸速率。

MT/s

MT/s = Mega-Transfers per second,是每條 DQ line 每秒的資料傳輸次數。DDR 雙邊緣下 MT/s = 2 × DQS clock(MHz)。換成頻寬:BW = MT/s × bus width。所以 1600 / 2400 MT/s 就是 Toggle / ONFI 的 speed grade = IO bandwidth,跟 program / read 的真實媒體速度是兩件事。

FA 陷阱

若你環境裡的「CTL 頻率」指的是驅動 DQS 的 toggle clock,那調它就是在調 IO bandwidth(同時也在改 timing margin,見第 09 章)。但若「CTL」指的是 array 端某個 internal clock(例如影響 tR / sensing 的),那調的是 array timing,意義完全不同。先確認 CTL 的定義,再判斷你動到的是哪一塊頻寬。

Array op 04

Read:把 cell 的 Vth 翻譯成 0/1

Read 分兩段:array sense(tR,慢) 把整個 page 讀進 cache register,接著 data out(快) 用 DQS strobe 把資料搬出去。命令序列(opcode 為通用示例,實際依 vendor):

00h [col0 col1 row0 row1 row2] 30h → R/B# low (tR) → Dout × N (DQS)
圖 3. Read:30h 後 R/B# 拉低跑 tR(array sensing,真實 FLH 速度);完成後 DQS strobe 把整個 page 高速搬出(IO 速度)。
機制重點 · sensing

讀取本質是「在某個 read level(Vread / Vref)下,看 cell 導不導通」。selected WL 給 Vread,其餘 unselected WL 給 Vpass(夠高讓它們無論狀態都導通),BL 預充後看放電與否判斷 Vth 在 read level 哪一側。TLC/QLC 一個 page 要多個 read level、甚至 multi-strobe,這就是 tR 隨 bit/cell 增加而拉長的原因。

Senior 思維 · read 連到 distribution

read level 落在兩個相鄰 Vth 分佈之間的 valley。RBER 的本質是 valley 兩側的分佈尾巴重疊。retention 使分佈下移、read disturb 使被讀 WL 鄰居上移、temperature 改變分佈寬度 —— 任何讓 valley 變淺的因素都直接抬高 RBER。看 read fail,先想「是哪個 valley、哪一側尾巴吃進來」。

Array op 05

Program:用 ISPP 把電子一步步打進去

Program 序列:資料先灌進 cache register,10h confirm 後 array 進入 program algorithm。

80h [col0 col1 row0 row1 row2] Din × N (DQS) 10h → R/B# low (tPROG)
圖 4. 注意 data-in 期間 WE# 保持 high、改由 DQS strobe(DDR)。10h latch 後 R/B# 拉低跑 ISPP / FN tunneling —— 這段就是 tPROG(真實 FLH 速度,圖中嚴重未按比例)。
機制重點 · ISPP 與 incremental step pulse

Program 用 ISPP(Incremental Step Pulse Programming):打一個 program pulse(FN tunneling 把電子注入 storage layer)→ verify(讀回 Vth)→ 沒到目標就把 Vpgm 抬高一個 ΔVpgm 再打,直到通過或用盡 max loop。step 越小、分佈越窄,但 tPROG 越長。tPROG 與 Vth distribution 寬度是一組 trade-off,不是獨立參數。

FA 陷阱 · 反直覺的 program disturb 方向

在 3D NAND CTF,program disturb 可能造成 Vth 下移(soft erase):強 channel boosting 在 inhibit cell 形成反向電場,把電子拉出。這跟 read disturb 造成 Vth 上移的方向相反。判 disturb 時,先確認方向,別預設「disturb 一定讓 Vth 上升」。

Senior 思維 · 連到 yield

tPROG 拉長 = ISPP loop 變多 = cell 越來越難打進去,通常代表 oxide 老化(P/E 累積)或 cell-to-cell 差異變大。program loop count 是極佳的早期 endurance 指標,比等到 program fail 才看更早。把 loop count 分佈納入 inline monitor,能在 yield 掉之前就看到趨勢。

Array op 06

Erase:整個 block 一次清空

Erase 是三大操作裡唯一以 block 為單位、也是最慢(tBERS 約 ms 等級)的。序列極簡:

60h [row0 row1 row2] D0h → R/B# low (tBERS)
圖 5. Erase 命令最短,但 tBERS 最長。因為太慢,multi-plane erase 幾乎是標配。
機制重點 · FN erase 與 erase 方向

Erase 用 FN tunneling 把電子從 storage layer 抽出(2D 是 substrate 加高壓,3D GAA 結構則靠 gate-induced 機制),讓整個 block 的 cell Vth 一起被打到 erase state(最低 Vth)。erase 也用類似 ISPP 的 erase-verify loop。erase 是對 oxide 應力最大的操作,endurance 的「P/E cycle」主要由 erase 應力主導。

Parallelism 07

Interleave:把慢 array 藏在快 IO 後面

單顆 die 的 FLH BW 很低。系統靠兩個維度平行化:die 內 multi-plane(同 die 多 plane 同步動作),與 channel 上 multi-die interleave(資料載完一顆就換下一顆,讓 bus 不閒置)。

圖 6. 同一段 wall-clock,藍 tDL 一格接一格(channel 100% 利用 = IO 在做事);amber tPROG 在時間軸重疊、被藏起來。系統合成 throughput ≈ N 顆並行。

balance point — 設計的甜蜜點

讓 channel 永不 idle 的條件:

tPROG ≈ (N − 1) × tDL   # N = 並行 die 數

當 Die0 的 program 剛好做完(虛線處),最後一顆也剛好載完 → bus 立刻回頭重載 Die0,從不空等。這就是經典的 array access time ≈ N × data transfer time 平衡。

Senior 思維 · 兩種失衡

tPROG 太長 / die 太少 → array-bound:bus 載完 N 顆後空等 program,channel idle,throughput 被 FLH 卡死。tPROG 太短 / die·channel 太多 → IO-bound:DQ bus 飽和,再加 die 也沒用,此時才靠提高 DDR rate 解。量到 sustained BW 卡住時,先判斷貼近哪一邊。

Parallelism 08

Cache 操作:在同一顆 die 內也做流水線

Interleave 是跨 die;cache 是在單顆 die 內、用 cache register + data register 兩級緩衝,讓「IO 傳輸」與「array 動作」在同一顆 die 上重疊。

Cache program(15h 取代 10h)

下達 cache program(以 15h confirm)後,資料從 cache register 複製到 data register,array 開始寫上一筆;同時 controller 已可把下一筆資料灌進 cache register。tDL(n+1)藏進 tPROG(n) 裡。

圖 7. Cache program:load D(n+1) 與 program D(n) 同時進行。對「連續大量寫入」的 throughput 提升極明顯。

Cache read(31h / 3Fh 類)

對偶概念:array 把下一個 page sense 進來的同時,controller 正把上一個 page 從 register 搬出。把 tR(n+1) 藏進 Dout(n) 裡,連續 sequential read 幾乎只剩 IO 時間。

機制重點 · 為什麼需要兩級 register

cache register 面向 IO、data register 面向 array。資料先進 cache register,複製到 data register 後 array 才開始動,於是 cache register 立刻空出來接下一筆。沒有兩級緩衝,IO 與 array 就無法在同一顆 die 上 overlap。

Senior 思維 · interleave vs cache 何時用

Cache 解決「單 die 連續操作」的 overlap;interleave 解決「跨 die 並行」的 throughput。兩者正交、可疊加。但 cache 吃較多 register 與 power,且 cache program 中途出錯的 error handling 較複雜(上一筆已寫、這一筆還在 register)。做 FA 時,cache 模式下的 fail 要特別釐清是哪一筆、在哪一級 register。

IO problem 09

IO 問題的核心:UI、eye 與 timing margin

IO 跑越快不是免費的。提高頻率 = 縮短 UI(unit interval),但 device 的 skew / jitter / ISI 在絕對時間(ps)上幾乎固定,於是它們吃掉 UI 的比例變大,可用的 eye(資料有效窗)被壓窄。

UI = 1 / (2·f)  # DDR 一個 bit 的時間
eye ≈ UI − (tDQSQ + jitter + ISI)  # 後者固定 ps,f↑ 不變

互動:拉動 CTL 頻率,看 eye 收縮

同一顆 die、固定 skew budget(~0.32 ns)。把頻率往上拉,看 UI 縮短、margin 變窄,直到 eye 關閉(latch fail)。

CTL freq PASS
MT/s 1600 ×8 BW 1.60 GB/s UI 0.625 ns eye 0.305 ns margin 49%
機制重點 · scan / shmoo 與頻率的耦合

scan window 本身不是頻寬,它在量 timing margin(eye)。掃 sampling phase(DQS / RE# delay)、常再加 Vref,得到通過區。調 CTL 頻率 = 調 IO bandwidth;而頻率↑ → UI↓ → 固定 skew 吃更高比例 → scan window 變窄。scan 就是讓你「看見」這個 trade-off 的工具。

FA 陷阱

看到 transfer / CRC error,先看它對頻率的反應:隨頻率惡化、降頻就消失 → IO / SI-bound(loading、skew、Vref、ODT),不是 cell。別把 IO margin 不足誤判成 cell 退化 —— 這是新人最常踩的坑。

IO problem 10

什麼決定一顆 die 的 IO 上限?

一顆 die 能跑多快的 ONFI 速度,本質由 die 上的 peripheral IO 電路 + process 決定,跟 cell array 無關(因為 cache register 解耦)。但「實際跑得到 grade」是 die、channel、controller 的 co-design。

on-die 電路因素(設定能力上限)

區塊影響
Output driver / padslew rate、drive strength;pad capacitance、ESD 結構(電容大 → 拉低上限)
ODTon-die termination,高速必備,吸收反射;沒 ODT 上不了高 grade
Zq calibrationdriver / ODT 阻抗對齊 channel

Periphery CMOS / processdevice fT、寄生、on-die power delivery / decap、serializer prefetch 寬度

系統因素(決定實際達成值)

Channel loading:每多掛一顆 die/CE,bus 多一份 pad cap → 上限往下掉(所以 grade 常標「up to X MT/s @ N CE/channel」)。PCB:trace 長度、阻抗、stub、crosstalk、反射。Controller PHY:RX/TX、DLL,以及 training(Vref training、read/write leveling、per-lane deskew、ODT)。die 再快,PHY 不 train 也達不到 grade。

機制重點 · IO power 的代價

P_IO ∝ η · f · C · V²。頻率越高 IO 功耗越大;在 multi-die / multi-channel 同時 active 時,這是真實的系統上限,常逼著降頻或降 Vccq。

Senior 思維 · 怎麼測 die 的本質 IO 能力

一顆 rated 2400 的 die 在系統上跑不到 2400,先別當成 die 壞:(1) 降到 1 CE/channel、開 ODT/training、用乾淨短 channel,測 intrinsic IO capability;(2) 若這樣就過 → 問題在 loading / PHY / SI;(3) 若降載 + training 都做了還過不了 → 才是 die-level IO 電路 / process 問題(driver 弱、Zq/ODT cal fail、DQ-DQS skew 超規)。

Senior methodology 11

FA 方法論:先分清楚是哪一塊壞

這是整本手冊的收束。Senior 與新人的差別,不在記得多少 spec,而在面對 fail 時能不能在三十秒內把問題歸到正確的 domain,然後只往那條路深挖。

圖 8. 第一刀永遠是「error 隨頻率變化 vs 隨 P/E/retention/溫度變化」。先固定保守頻率掃出乾淨 eye 確認 interface 沒問題,再去談 array RBER —— 否則會把 IO margin 不足誤判成 cell 退化。

Senior 思維 · 把 device 連到 distribution 再連到 yield

新人停在「這顆 cell 怎麼了」;senior 一定走完三層:device 行為 → Vth distribution 後果 → yield / reliability 衝擊。例如「tPROG 變長」不只是慢,而是 ISPP loop↑ → 分佈尾巴變寬 → read valley 變淺 → RBER↑ → ECC 餘裕吃緊 → 最終 yield/壽命掉。能把這條鏈條講完整,就是 senior。

核心心法
