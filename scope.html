01 — FUNDAMENTALS

示波器到底是什麼?

一句話:示波器是「把電壓隨時間變化畫成圖」的儀器。三用電表只告訴你「現在幾伏特」,示波器則畫出整段「電壓的形狀」——這就是為什麼看得到雜訊、毛刺(glitch)、上升邊緣、與訊號是否乾淨。

難度分級 新手開機就該會的基礎 資深能獨立量得準的工程師 專家Flash/SSD 合規與校準

縱軸 Y

電壓

螢幕往上往下代表電壓高低。一格的大小叫每格電壓 (V/div),你可以縮放它。

新手
橫軸 X

時間

螢幕左到右代表時間流動。一格時間叫時基 (s/div),決定你看到的是 µs 還是 ns 等級。

讓重複的波形「定住」在同一個位置不亂跑的關鍵。沒有觸發,畫面只會是一團模糊。

把電路上的訊號接進示波器的橋樑。選錯探棒,再好的示波器也量不準(後面有專章)。

02 — LEARNING PATH

從小白到專家的三個階段

不用一次學完。照這條路走:先會「看懂畫面」,再會「設對參數」,最後才碰 Flash/SSD 的合規量測。點下面切換階段。

1

認得四個旋鈕區

垂直(V/div + 位置)、水平(時基 + 位置)、觸發(準位 + 邊緣)、與 Autoset 自動鍵。先按 Autoset 讓波形跳出來,再手動微調。

2

接探棒 + 做補償

把附的被動探棒接上,夾到示波器的「方波校準」端子,轉探棒上的小螺絲,讓方波變成漂亮的直角(probe compensation)。沒補償,量出來的邊緣是錯的。

3

讀出 Vpp 與頻率

用內建自動量測(Measure)讀峰對峰電壓、頻率、週期。先相信儀器,再學會自己用「格數」驗算。

4

把觸發準位調穩

波形如果一直左右亂跑,八成是觸發沒設好。把觸發準位拉到波形中間、選對邊緣,畫面就定住了。

1

頻寬與取樣率怎麼選

記住兩條業界拇指法則:頻寬 ≥ 訊號最高頻率的 3~5 倍;取樣率 ≥ 頻寬的 2.5~5 倍。下一節有互動計算機。

2

看懂「混疊 (aliasing)」

取樣率不夠時,高頻訊號會被「畫成」一條根本不存在的低頻波。這是新手最常踩、又最難察覺的坑。下面有 demo。

3

升級到主動 / 差動探棒

被動探棒約 500 MHz 就到頂、且會對電路造成負載(loading)。量高速訊號要換主動或差動探棒。

4

進階觸發 + 解碼

用脈寬觸發、邏輯觸發抓特定事件;用 MSO 的數位通道 + 序列解碼看匯流排(I²C/SPI/匯流排命令)。

1

建立 / 保持時間 (setup/hold)

NAND 的時序根基:資料 (DQ) 必須在時脈邊緣前穩定「建立時間」、邊緣後維持「保持時間」。違反就是讀寫錯誤的源頭。

2

DQ / DQS 與眼圖

高速 Toggle/ONFI 模式下,用 DQ(資料)對 DQS(資料閘控)做眼圖,一眼看出 jitter、ISI、雜訊裕度。

3

ONFI 合規量測

用 Tektronix TEK-PGY-ONFI 或 Keysight 邏輯分析儀方案,自動量 SDR / NV-DDR / NV-DDR2/3 模式對 ONFI 規格的時序合規。

4

探測策略 (probing strategy)

DQ 是共用匯流排、CE/ALE/CLE/RE/WE 是各自獨立訊號。用 interposer 轉接 + 高頻寬探棒,規劃要量哪幾條線才量得到、又不破壞訊號。

資料來源 Tektronix — ONFI Electrical Timing Analysis Keysight — Oscilloscope Basics
03 — KEY SPECS

五個你一定要懂的規格

買示波器、設參數,90% 的決定都繞著這五個數字。每個都附泰克 / Keysight 官方說法。

新手
BANDWIDTH

頻寬

示波器能準確量到多高頻率的能力。定義是:輸入正弦波被衰減到真實振幅 70.7%(−3 dB) 時的頻率。超過頻寬,波形會越畫越小、越失真。

拇指法則(Keysight):頻寬至少要 訊號基頻的 3~4 倍,才能保住第 1、3 次諧波,把方波畫得像方波。
Tek — Evaluating OscilloscopesKeysight KB
資深
RISE TIME

上升時間

示波器能捕捉多快的邊緣變化。對數位訊號(脈衝、邊緣)比頻寬更直接相關。經典關係式:

頻寬 × 上升時間 ≈ 0.35(傳統)→ 0.45(高階即時示波器)
高階即時示波器因為過取樣後濾波較陡(防混疊),係數會接近 0.45。
Tek — Bandwidth vs Rise Time
新手
SAMPLE RATE

取樣率

每秒抓幾個點(GS/s)。根據奈奎斯特定理,取樣率至少要訊號最高頻率的 2 倍才不會混疊;實務上建議 3~5 倍頻寬才能把快速邊緣畫準。

取樣率不夠 → 混疊 → 畫出一條根本不存在的假波形。這是最危險的陷阱(下面可實測)。
Keysight — Sample RateTek — Real-time vs Equiv-time
資深
RECORD LENGTH

記錄長度 / 記憶體深度

一次能存多少個取樣點。記憶體越深,就能在「高取樣率」下還保有「長時間」的擷取——兩者通常互相拉扯。

注意:記憶體填滿時取樣率會自動下降;太深的記憶體也會拉長死區時間(dead time)、降低更新率。
例:Tek 4 Series MSO 32.5 Mpts vs Keysight 4000A X 4 Mpts(各廠定位不同)。
Keysight — Memory DepthTek — 4 Series vs 4000A
資深
WAVEFORM CAPTURE RATE

波形更新率(抓毛刺的能力)

每秒能畫幾次波形(wfms/s)。更新率越高、死區時間越短,越容易抓到偶發的毛刺與異常。這是「能不能看到罕見事件」的關鍵——光看頻寬不夠。Tek 強調 FastAcq、Keysight 強調高 update rate,兩家各有做法。

04 — DON'T BE FOOLED

實測「混疊」:示波器怎麼騙你

✓ 取樣充足

奈奎斯特要求取樣率 ≥ 2× 訊號頻率。一旦低於 2×(紅色警告),黃色重建波就會「假裝」成不存在的低頻訊號——你在螢幕上看到的,根本不是真的。

05 — PROBES

探棒:最容易被忽略的精度來源

泰克有句話:量測從探棒尖端就開始了。再貴的示波器,配錯探棒就量不準。三種主力探棒一次看懂。

新手
PASSIVE · 被動

被動探棒

最常見、最便宜、最耐用,通常隨機附贈。高阻抗版頻寬約到 500 MHz 為止;低阻抗版又可能造成不能接受的直流負載。1:1 被動探棒頻寬常 < 35 MHz。

用前一定要做補償(compensation):接方波校準端、轉螺絲讓方波變直角。
資深
ACTIVE · 主動

主動探棒

探頭內含放大器/緩衝器,需供電。最大優點:極低輸入電容、在寬頻範圍內對電路負載極小 → 量高速訊號更準。代價是貴、且脆弱要小心拿。

高速 NAND(NV-DDR2/3、Toggle)的時序量測通常需要主動或差動探棒。
專家
DIFFERENTIAL · 差動

差動探棒

兩條探針量「兩點之間的電壓差」,兩端都不接地。對共模雜訊免疫,適合在吵雜環境量低準位訊號——正是 DQS 這類差動閘控訊號的首選。

ONFI 高速模式建議用類比通道 + 主動/差動探棒,動態範圍與訊號保真度最好。

🔌 負載效應 (loading):探棒接上去那一刻,本身的電容/電阻會「偷走」一點訊號、改變你想量的電路。探棒電容越低、對高速訊號的干擾越小——這就是主動探棒值錢的地方。

資料來源 Tektronix — ABCs of Probes Keysight — Probe Selection Guide Keysight — Passive vs Active
06 — HOW TO USE

實機操作步驟:從開機到讀數

第一次拿到示波器,照這 9 步做一遍就會了。點下面任一步驟,上方面板會幫你指出對應的旋鈕/按鍵,做完就能在下一節的模擬器上練。

👆 點下面任一步驟,面板上對應的旋鈕/按鍵會亮起。
1

開機 + 選通道▸ 電源 + CH1

開電源,按下要用的通道鍵(CH1)讓它亮起。多數訊號用一個通道就夠。

2

接探棒 + 做補償▸ PROBE COMP

探棒接到 CH1,夾到面板的「方波校準(PROBE COMP)」端子,轉探棒上的補償微調,讓方波頂部變成乾淨直角。沒補償,量到的邊緣是錯的(下方有互動)。

3

探頭接到待測點▸ CH1 IN

探針鉤住訊號點、鱷魚夾接電路接地。地線越短越好,否則高速訊號會多出振鈴(ringing)。

4

按 Autoset 讓波形跳出來▸ AUTOSET

找不到波形時先按 Autoset,示波器會自動把垂直/水平/觸發設到看得到波形的範圍。先有畫面,再手動微調。

5

調垂直 V/div + 位置▸ VERTICAL

轉「每格電壓(V/div)」讓波形上下約佔 6 格(別超出螢幕),再用垂直位置鈕移到中間。佔越多格、讀數越準。

6

調水平時基 s/div▸ HORIZONTAL

轉「時基(s/div)」讓螢幕剛好顯示 2~3 個週期——太擠看不清邊緣,太鬆看不到細節。

7

設定觸發:邊緣 + 準位▸ TRIGGER

觸發源選 CH1、類型選「邊緣」,把「觸發準位」轉到波形高低點中間。畫面一定住代表觸發成功;一直左右亂跑,八成是準位拉到波形範圍外。

8

開自動量測讀數▸ MEASURE

按 Measure 加入要看的量測:Vpp、頻率、週期、上升時間。先相信儀器,再用「格數 × 每格值」自己驗算。

9

(進階)存波形 / 截圖▸ SAVE

用 Save/Recall 或截圖把畫面與設定存到 USB,方便做報告或事後比對。

WAVEFORM · 訊號類型

波形種類

正弦 Sine wave:只有單一頻率的交流訊號。方波 Square wave:在高/低電位間切換,含基頻 + 大量奇次諧波,常用來看上升邊緣與頻寬。脈衝 Pulse:短暫的突波,用來抓 glitch / 單次事件。

TIME BASE · s/div

時基

水平方向「每一格」代表多少時間。整個螢幕的時間窗 = s/div × 10 格。轉小 → 看更快的細節。

VOLTS/div · V/div

每格電壓

垂直方向「每一格」代表多少電壓。垂直總量程 = V/div × 8 格。縮放它把波形剛好塞進螢幕。

TRIGGER LEVEL

觸發準位

觸發用來比較的「電壓門檻」。當訊號(依設定的邊緣方向)穿越這個準位時,示波器就對齊重畫一次,波形才會定住不亂跑。

Vpp · PEAK-TO-PEAK

峰對峰電壓

波形最高點到最低點的電壓差:Vpp = Vmax − Vmin。代表訊號的整體振幅大小。

FREQUENCY · Hz

頻率

訊號每秒重複幾個週期,單位赫茲 Hz。與週期互為倒數:f = 1 / T。

PERIOD · s

週期

一個完整波形重複一次所花的時間。與頻率互為倒數:T = 1 / f。

TRIGGER STATUS

觸發狀態

穩定 Triggered:條件滿足、波形鎖定不動。失去觸發 Untriggered:條件沒滿足(例如準位拉出波形範圍),畫面開始漂移亂跑。

圖 A · Legacy / SDR 介面控制時序(命令 → 位址 → 命令 → 資料輸出)

圖 B · DDR DQ / DQS 與建立/保持(可切換 訓練前 / 訓練後)
DQS 資料閘控(strobe) DQ 資料窗 (D0…D7) tDS / tDH 建立 / 保持

怎麼讀:高速模式改用 DQS 閘控,每個 DQS 邊緣(↑與↓)各閂一筆 DQ → 這就是 DDR 雙倍資料率。資料必須在 DQS 邊緣前穩定(tDS 建立)、邊緣後維持(tDH 保持)。訓練(DQ training)的目標就是把 DQS 邊緣置中在資料窗,讓 setup/hold 裕度最大——對應名詞:DQ/DQS、建立/保持時間、眼圖、DQ 訓練、有效資料窗。

09 — FLASH / SSD

Flash / SSD 實戰量測

這是你工作真正要碰的部分。NAND 介面速度越來越快、電壓越來越低,訊號完整性(SI)成了良率的關鍵戰場。

資深
SETUP / HOLD

建立時間 / 保持時間

資料 (DQ) 必須在時脈/閘控邊緣之前就穩定(建立時間),並在邊緣之後維持一段時間(保持時間)。任何一邊違反,接收端就抓到錯的位元——這是 NAND 讀寫失敗最根本的時序原因。

資深
DQ / DQS

資料與資料閘控

DQ 是資料線(讀寫共用的匯流排),DQS 是資料閘控訊號。寫入時主機驅動 DQS、讀取時 NAND 驅動 DQS;每個 DQS 的上升與下降邊緣都用來閂存一筆資料。量測重點就是兩者的相對時序關係。

👁️ 眼圖互動 Demo

眼圖 = 把成千上萬個位元疊在一個位元週期(UI)上。眼睛張得越開,訊號越乾淨。拉動下面的「抖動+雜訊」滑桿,看眼睛怎麼閉起來。

✓ 眼開,裕度充足

眼高 (eye height) = 訊噪比/電壓裕度;眼寬 (eye width) = 時序裕度。閉合的眼代表 jitter、ISI(碼間干擾)或雜訊太大,接收端容易誤判 → 提高位元錯誤率 (BER)。

專家
ONFI COMPLIANCE

ONFI 合規量測方案

ONFI 是並列匯流排,含控制訊號、時脈與資料。高速(DDR2/3)合規量測建議用示波器的類比通道(動態範圍大、取樣率高、保真度優於數位通道)。

Tektronix TEK-PGY-ONFI:自動量 Command / Address / Data-in / Data-out,支援 SDR、NV-DDR、NV-DDR2/3;Detail View 把每筆電氣量測標註回類比波形,方便除錯;提供 DQ/DQS 眼圖與眼高/眼寬游標。跑在 MSO/DPO70000 系列(4–33 GHz),搭 P7500/P7300 探棒。

Keysight 16860A 系列邏輯分析儀:配 10 GHz quarter-channel timing 選項可擷取 ONFI 流量,再用 B4661A 記憶體分析軟體做效能分析。

🧭 探測策略很關鍵:DQ 是所有 NAND die 共用的匯流排,而 CE、ALE、CLE、RE、WE 是各自獨立的訊號。驗證工程師要先規劃「用 interposer 轉接、量哪幾條線」,才能拿到準確量測又不破壞訊號完整性。

🎯 介面訓練與校準(高速 NAND 的關鍵)

當速率衝上 800 MT/s 以上,光靠固定時序不夠了。控制器必須在初始化時「訓練」介面——自動找出最佳取樣點與判定電壓。這三件事是良率與訊號完整性的核心。

專家
DQ TRAINING

DQ 訓練(讀/寫)

自 ONFI 4.1 起,在 ZQ 校準之上新增,專門對付 >800 MT/s、重負載的裝置。目的:把 DQS 邊緣的取樣點對齊到 DQ 資料眼圖的正中央。

讀訓練:調整讀取時序偏移,讓取樣點落在眼圖中心 C 點。
寫訓練:對齊主機送進 NAND 的 DQ。常再細分 bit-skew(對齊各 DQ 邊緣)+ centering(把 DQS 置中)。
專家
DC / VREF TRAINING

DC 訓練(Vref / DCC)

處理「電壓軸」的校準,跟 DQ 訓練的「時間軸」互補:

VrefQ 校準:調整 DC 判定電壓門檻,讓接收端在最佳準位切分 1/0(ONFI 5.x 進一步強化)。
DCC 占空比校正:修正高速下上升/下降邊緣不對稱(如 RE# 互補訊號)造成的工作週期失衡。
專家
SCAN WINDOW

掃描窗 / Shmoo

訓練怎麼「找到」中心?靠掃描:在不同取樣相位(與 Vref)下逐點讀寫、記 pass/fail,畫出有效資料窗。

找出窗的左右邊界(最小裕度寬度),取中點當取樣點。眼圖是「類比 SI 視角」,Shmoo 是同一件事的「pass/fail 裕度視角」。窗越大、良率越穩。

🔗 一條主線串起來:DQ/DQS 相對時序 → 用示波器看眼圖(SI) → 控制器跑 DQ training 把取樣點置中(時間軸)→ DC/Vref training 把判定電壓置中(電壓軸)→ 用 Shmoo 掃描窗驗證裕度。良率分析時,窗縮小或偏移,往往就是訊號完整性或訓練收斂出了問題。

🟩 Shmoo 掃描窗互動 Demo

綠色 = pass、紅色 = fail,中間的綠色開口就是「資料有效窗」。用滑桿移動取樣點(相位 × Vref),看它落在窗內還窗外;或按「自動訓練」把取樣點置中。把訊號裕度往下拉,看窗怎麼縮小。

✓ PASS · 取樣點在窗內

真實的訓練演算法就是在做這件事:掃出這張圖、找到 pass 區的中心、把取樣相位與 Vref 設到那裡。取樣點越靠近邊緣,溫度/電壓一漂就掉出窗 → 讀寫錯誤。

👁️ vs 🟩 眼圖 與 Shmoo 到底差在哪?

兩個都在看「訊號好不好、裕度夠不夠」,但量的東西完全不同:眼圖畫的是「訊號本身」,Shmoo 畫的是「測試結果」。

🧠 一句話分辨:眼圖是「把訊號本身畫出來」,反映類比品質(邊緣糊不糊、雜訊大不大);Shmoo是「把測試結果畫出來」,反映操作裕度,用來找置中的最佳取樣點。
🔗 關聯:眼圖開得越大,對應的 Shmoo 綠色窗通常也越大;DQ / Vref training 的目標就是把取樣點移到 Shmoo 窗正中央,也等於對到眼圖最開的地方。
